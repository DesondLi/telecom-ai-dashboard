# 电信AI数据治理系统 - 前端优化PRD

## 1. 项目背景

电信AI数据治理与用户画像分析系统是一个基于AI驱动的电信投诉用户数据分析平台，帮助电信客服人员快速分析投诉用户特征，识别风险用户，并提供智能化的处理建议。

当前系统功能已经完整实现，采用Streamlit框架开发为单页应用，但UI设计较为基础，存在视觉风格不统一、信息层次不清晰、交互体验不够友好等问题，需要进行前端优化，提升整体用户体验和专业度。

### 项目目标
- 提升视觉设计质量，打造专业级数据分析产品外观
- 优化信息层级结构，让关键信息一目了然
- 改善交互体验，提升客服人员操作效率
- 增强移动端适配，支持不同设备访问
- 保持原有功能完整性，不改变业务逻辑

## 3. 用户操作流程

### 当前用户操作流程

1. **进入系统** → 侧边栏显示控制面板
2. **选择数据源** → 系统加载对应数据（真实数据/模拟数据）
3. **选择用户** → 在下拉框中选择目标用户联系电话
4. **查看概览信息** → 主界面显示核心指标和详细信息
   - 第一行：四个核心指标卡片
   - 第二行左：投诉指标柱状图对比
   - 第二行右：客服行动标签+业务信息
5. **触发AI分析** → 点击按钮生成AI诊断/用户画像
6. **查看元数据** → 可展开查看数据血缘和原始数据

### 用户角色

- **客服人员**：日常使用，查询投诉用户信息，获取处理建议
- **客服主管**：分析高危用户，制定处理策略
- **数据分析师**：查看数据血缘，了解数据加工过程

## 4. 痛点识别

基于现有代码分析，当前UI/UX存在以下问题：

### 4.1 视觉风格问题
- **主题不明确**：缺少统一的品牌色彩和电信行业特色
- **视觉层次弱**：所有内容区块阴影和边距相似，难以快速区分主次
- **配色单调**：默认Streamlit配色，缺乏专业感
- **缺少呼吸感**：内容区域padding不足，显得拥挤
- **Logo和品牌缺失**：只有一个在线图标，缺少品牌标识

### 4.2 信息层级问题
- **侧边栏内容过多**：统计指标放在侧边栏占用空间，主空间受限
- **关键指标不突出**：高风险用户没有视觉高亮提示
- **标签样式不够鲜明**：虽然按风险分色，但样式不够突出
- **AI结果展示不区分**：AI分析结果和普通内容视觉上没有区隔
- **缺少内容分组**：相关信息没有视觉分组，整体性差

### 4.3 交互效率问题
- **用户选择不便**：用户量多时下拉框选择困难，缺少搜索功能
- **AI分析需要手动触发**：默认不显示，每次都需要点击按钮
- **没有用户快速切换**：选择新用户需要回到侧边栏下拉框
- **缺少筛选功能**：无法按风险等级、客户等级等维度筛选用户
- **页面滚动过长**：所有内容都在一个页面，需要多次滚动

### 4.4 可用性问题
- **缺少加载状态反馈**：除了AI分析外，数据加载没有明确进度提示
- **错误提示不友好**：数据加载失败直接显示错误，没有引导解决方案
- **移动端适配差**：Streamlit默认布局在小屏幕上体验不佳
- **缺少风险状态可视化**：高危用户没有醒目的视觉警示
- **帮助信息不足**：新用户对各功能含义不了解，缺少tooltip说明

## 5. 优化需求

### 5.1 必须优化 (P0)

| 需求ID | 需求描述 | 验收要点 |
|--------|----------|----------|
| P0-01 | **统一视觉主题**：设计电信风格主题，使用蓝色系为主色调，统一各组件样式 | 品牌色彩统一，符合电信行业属性 |
| P0-02 | **重构页面布局**：将统计概览移至主界面顶部，侧边栏只保留数据源和用户选择 | 释放主空间，信息更集中，层次更清晰 |
| P0-03 | **增强用户选择体验**：用户选择器添加搜索功能，支持快速定位用户 | 海量用户下也能快速查找，提升效率 |
| P0-04 | **强化风险可视化**：根据用户风险等级（高危/中危/低危）在页面顶部显示对应颜色的警示条 | 一眼识别风险等级，优先处理高危用户 |
| P0-05 | **优化标签展示**：改进客服行动标签样式，增大尺寸，强化颜色区分 | 不同风险标签视觉差异明显，易于识别 |
| P0-06 | **AI分析结果默认展示**：页面加载后自动展示AI分析结果，无需手动点击 | 减少用户操作步骤，打开页面即可看到完整分析 |
| P0-07 | **优化内容区块间距**：调整各个内容区块padding、margin、阴影，增强视觉层次 | 页面不拥挤，区块区分清晰 |
| P0-08 | **改进指标卡片设计**：核心指标卡片增强视觉设计，突出关键数值 | 指标一目了然，对比清晰 |

### 5.2 应该优化 (P1)

| 需求ID | 需求描述 | 验收要点 |
|--------|----------|----------|
| P1-01 | **添加用户筛选功能**：支持按风险等级、客户等级、投诉次数等维度筛选用户列表 | 可快速定位高价值目标用户 |
| P1-02 | **优化图表展示**：投诉指标图表改进配色，添加标题说明，优化坐标轴 | 更易读，对比更清晰 |
| P1-03 | **升级数据血缘展示**：保持graphviz可视化，改进卡片样式，添加说明文字 | 更易理解数据加工过程 |
| P1-04 | **添加页面锚点导航**：顶部添加快速导航栏，点击可跳转到对应区块 | 长页面快速定位，提升导航效率 |
| P1-05 | **改进业务信息展示**：使用更清晰的网格布局，增强信息可读性 | 业务信息结构清晰，查找方便 |
| P1-06 | **添加收藏/收藏功能**：支持将重要用户收藏，方便快速访问 | 高频用户无需重复搜索 |
| P1-07 | **优化按钮交互**：主要操作按钮（AI诊断、生成画像）增强视觉突出度 | 引导用户操作，减少学习成本 |

### 5.3 可以优化 (P2)

| 需求ID | 需求描述 | 验收要点 |
|--------|----------|----------|
| P2-01 | **添加最近浏览记录**：侧边栏显示最近浏览的用户，支持快速切换 | 提升多用户分析场景效率 |

| P2-04 | **移动端适配优化**：针对手机屏幕调整布局和字体大小 | 支持手机随时查看 |
| P2-05 | **添加使用说明**：在侧边栏添加功能说明和使用提示 | 降低新用户学习成本 |
| P2-06 | **暗黑模式支持**：支持深色/浅色主题切换 | 满足不同用户偏好 |
