## 1. 设计概述

### 1.1 设计目标

基于PRD和UX设计，为电信AI数据治理系统建立完整统一的视觉设计系统：

1. **打造专业企业级视觉风格** - 符合电信行业属性，专业可信
2. **建立统一设计规范** - 确保各组件样式一致，便于前端实现
3. **强化信息层级** - 通过色彩、字体、间距建立清晰视觉层次
4. **优化数据可视化** - 确保图表和数据展示清晰易读
5. **提升用户体验** - 通过良好的视觉设计引导用户操作

### 1.2 视觉风格定位

通过对同类企业级数据产品的调研分析，确定本产品视觉风格：

| 风格维度 | 定位说明 |
|---------|---------|
| **整体风格** | 专业、简洁、清晰、现代 |
| **色彩偏向** | 蓝色系主色调，符合电信行业属性 |
| **信息密度** | 适中，保证关键信息突出，保留呼吸感 |
| **视觉层次** | 清晰分明，通过阴影、间距、颜色区分层级 |
| **设计语言** | 圆角卡片、柔和阴影、干净留白、扁平化设计 |

**参考同类产品风格**:
- 阿里云DataWorks：清晰专业的企业级数据产品风格
- 腾讯云大数据：简洁现代的蓝色主题设计
- 华为云DataArts：结构化信息展示，层次分明
- 网易有数：良好的数据可视化设计语言

## 2. 色彩方案

### 2.1 主色调 - 电信蓝体系

采用蓝色系作为主色调，符合电信行业属性，传递专业、可信、科技感：

| 色阶 | 色值 | 使用场景 |
|-----|------|---------|
| **主色-深蓝** | `#0F51B8` | 品牌Logo、主按钮、高亮文字、边框强调 |
| **主色-标准** | `#1976D2` | 标题、链接、导航高亮、进度条 |
| **主色-浅蓝** | `#64B5F6` | 标签背景、进度条底色、次要按钮边框 |
| **主色-超浅蓝** | `#E3F2FD` | AI区块背景、表格斑马线、提示背景 |
| **主色-极浅蓝** | `#FAFCFE` | AI区块背景、卡片强调背景 |
| **主色-深蓝黑** | `#0D47A1` | 标题文字、强调文字 |

**色彩选择理由**:
- 蓝色是电信行业传统主色，用户已有认知基础
- 蓝色传递信任感和专业性，适合企业级产品
- 蓝色与警示色（红/橙/黄/绿）对比度良好，不冲突

### 2.2 功能警示色 - 风险等级体系

基于风险等级设计清晰的警示色彩体系，确保用户一眼识别：

| 风险等级 | 背景色 | 文字色 | 边框色 | 使用场景 |
|---------|-------|-------|-------|---------|
| **高危** | `#FFEBEE` | `#B71C1C` | `#FFCDD2` | 风险警示条、高危标签 |
| **中危** | `#FFF3E0` | `#E65100` | `#FFE0B2` | 风险警示条、中危标签 |
| **低危/关注** | `#FFFDE7` | `#F57F17` | `#FFF59D` | 风险警示条、关注标签 |
| **正常/低风险** | `#E8F5E9` | `#2E7D32` | `#C8E6C9` | 风险警示条、正常标签 |
| **信息提示** | `#E3F2FD` | `#1565C0` | `#BBDEFB` | 信息提示框、帮助提示 |

### 2.3 辅助功能色

| 颜色名称 | 色值 | 使用场景 |
|---------|------|---------|
| **成功色** | `#4CAF50` | 成功提示、确认按钮、正常状态 |
| **警告色** | `#FF9800` | 警告提示、待处理状态 |
| **错误色** | `#F44336` | 错误提示、错误状态、删除操作 |
| **链接色** | `#1976D2` | 超链接、可点击文字 |
| **文字链接hover** | `#0D47A1` | 链接悬停状态 |

### 2.4 中性色体系

建立清晰的中性色阶梯，用于文本、背景、边框：

| 颜色名称 | 色值 | 使用场景 |
|---------|------|---------|
| **背景色-页面** | `#F5F7FA` | 页面整体背景 |
| **背景色-卡片** | `#FFFFFF` | 卡片背景、模块背景 |
| **背景色-区块** | `#FAFBFC` | 次级区块背景 |
| **边框-分隔线** | `#E5E6EB` | 普通边框、分隔线 |
| **边框-浅色** | `#F0F0F0` | 浅色边框、内部分隔线 |
| **文字-一级标题** | `#1D2129` | 主标题、重要数据 |
| **文字-二级标题** | `#272E3B` | 区块标题、卡片标题 |
| **文字-正文** | `#4E5969` | 正文内容、普通文字 |
| **文字-辅助文字** | `#86909C` | 辅助文字、注释、占位符 |
| **文字-提示文字** | `#C9CDD4` | 次要提示、禁用文字 |
| **遮罩层** | `rgba(0, 0, 0, 0.45)` | 弹窗背景遮罩 |

### 2.5 色彩使用规范

1. **主色使用原则**
   - 主色占比控制在页面的10%-20%
   - 用作品牌识别和操作引导，避免大面积使用
   - 导航、按钮、链接使用主色，突出交互元素

2. **风险色使用原则**
   - 只用于风险相关内容，不滥用
   - 警示条整行使用浅色背景+深色文字
   - 标签使用浅色背景+深色文字，保持清晰

3. **中性色使用原则**
   - 页面背景使用中性浅色，减少眼睛疲劳
   - 文字遵循清晰的阶梯，确保良好可读性
   - 最小对比度保持在4.5:1以上（WCAG AA标准）

## 3. 排版规范

### 3.1 字体体系

**字族选择**:
- 优先使用系统默认无衬线字体，保证跨平台一致性和性能
- 中文：`-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", "Helvetica Neue", Helvetica, Arial, sans-serif`
- 英文/数字：`SFMono-Regular, Menlo, Monaco, Consolas, "Liberation Mono", "Courier New", monospace`

**字体加载**:
- 使用系统字体，无需额外加载，保证性能
- Streamlit环境下沿用其默认字体栈，在此基础上调整字号

### 3.2 字号阶梯

采用8px基准的字号阶梯，保持比例和谐：

| 规格 | 字号 (px) | 字重 | 行高 | 使用场景 |
|-----|----------|------|------|---------|
| **超大标题** | 32 | 700 | 1.3 | 页面主标题（极少使用） |
| **大标题** | 24 | 700 | 1.4 | 主标题、统计大数字 |
| **区块标题** | 18 | 600 | 1.4 | 卡片标题、区块标题 |
| **子标题** | 16 | 600 | 1.5 | 子区块标题、卡片子标题 |
| **正文-大** | 15 | 400 | 1.6 | 重要正文、AI分析正文 |
| **正文-常规** | 14 | 400 | 1.6 | 普通正文、标签文字、按钮文字 |
| **辅助文字** | 13 | 400 | 1.5 | 辅助说明、次要信息 |
| **小提示** | 12 | 400 | 1.5 | 脚注、提示文字、页码 |
| **超小文字** | 11 | 400 | 1.4 | 标签小文字、版权信息 |

### 3.3 字重使用规范

| 字重 | 数值 | 使用场景 |
|-----|------|---------|
| **Bold** | 700 | 主标题、核心指标数值、重要强调 |
| **Semi-bold** | 600 | 区块标题、子标题、按钮文字 |
| **Medium** | 500 | 标签文字、导航文字、次要强调 |
| **Regular** | 400 | 正文、辅助文字、普通内容 |

### 3.4 行高与间距规范

- **标题行高**: 1.3 - 1.4，保持紧凑
- **正文行高**: 1.6，保证良好可读性
- **段落间距**: 1em，段落之间保持适当间距
- **字间距**: 0px，默认字间距即可满足需求

## 4. 间距与布局规范

### 4.1 基础间距单位

采用8px栅格系统，所有间距都是8的倍数，保持一致性：

| 单位 | 像素值 | 使用场景 |
|-----|-------|---------|
| **xs** | 4px | 元素内间距、小图标文字间距 |
| **sm** | 8px | 紧凑间距、小组件间距 |
| **md** | 16px | 默认间距、卡片内边距、元素间距 |
| **lg** | 24px | 区块间距、卡片外边距 |
| **xl** | 32px | 大区块间距、页面顶部底部边距 |
| **xxl** | 48px | 页面大区块间距 |

### 4.2 页面整体间距

- **页面左右边距**: 16px (md)
- **区块之间间距**: 24px (lg)
- **区块内边距**: 20px (2.5 × 8px)
- **卡片之间间距**: 16px (md)
- **卡片内边距**: 20px

### 4.3 响应式断点

响应式设计断点与UX设计保持一致：

| 断点范围 | 布局适配 |
|---------|---------|
| **≥ 1920px** | 大屏桌面，充分利用空间，保持双列布局 |
| **1200px - 1919px** | 普通桌面，保持双列布局 |
| **768px - 1199px** | 平板，双列转单列 |
| **< 768px** | 手机，侧边栏可折叠，全部单列 |

### 4.4 网格布局规范

**整体统计概览卡片网格 (5个指标)**:

| 屏幕宽度 | 网格布局 |
|---------|---------|
| ≥ 1200px | 5列等宽 |
| 768px - 1199px | 3列 |
| < 768px | 2列 |

**核心指标卡片网格 (4个指标)**:

| 屏幕宽度 | 网格布局 |
|---------|---------|
| ≥ 1200px | 4列等宽 |
| 768px - 1199px | 2列 |
| < 768px | 2列 |

**双列内容区 (指标对比 + 标签业务)**:

| 屏幕宽度 | 布局 |
|---------|------|
| ≥ 1200px | 1:1 双列 |
| < 1200px | 单列 |

**AI分析区 (投诉诊断 + 用户画像)**:

| 屏幕宽度 | 布局 |
|---------|------|
| ≥ 992px | 1:1 双列 |
| < 992px | 单列 |

## 5. 组件设计规范

### 5.1 卡片组件

卡片是最基础的内容容器，所有内容区块都使用卡片：

**设计规范**:

| 属性 | 数值 | 说明 |
|-----|------|---------|
| **背景色** | `#FFFFFF` | 纯白背景 |
| **圆角** | 12px | 柔和圆角，现代风格 |
| **边框** | 无 | 纯阴影分层 |
| **阴影** | `0 2px 8px rgba(0, 0, 0, 0.06)` | 普通卡片阴影 |
| **强调阴影** | `0 4px 16px rgba(0, 0, 0, 0.08)` | 核心指标卡片阴影 |
| **内边距** | 20px | 内容与边框间距 |
| **外边距(底部)** | 24px | 卡片与下方区块间距 |

**卡片类型**:

1. **普通卡片** - 用于一般内容区块，使用普通阴影
2. **强调卡片** - 用于核心指标，使用稍重阴影，突出层次
3. **AI区块卡片** - 使用极浅蓝色背景 `#FAFCFE` + 浅蓝色边框 `#E3F2FD`
4. **警示条卡片** - 全宽，高度48px，根据风险等级改变背景色和文字色

**示例代码结构** (HTML参考):

```html
<!-- 普通卡片 -->
<div class="card card-normal">
  <h3 class="card-title">区块标题</h3>
  <div class="card-content">
    <!-- 内容 -->
  </div>
</div>

<!-- AI卡片 -->
<div class="card card-ai">
  <h3 class="card-title">🤖 AI分析结果</h3>
  <div class="card-content">
    <!-- 内容 -->
  </div>
</div>
```

### 5.2 指标卡片组件

用于展示统计指标和核心数据，分整体统计概览和核心指标两种：

**整体统计概览卡片设计**:

| 元素 | 规范 |
|-----|------|
| **容器** | 普通卡片，内边距16px |
| **指标名称** | 13px, `#86909C`, 400字重 |
| **指标数值** | 24px, `#1D2129`, 700字重 |
| **间距** | 名称与数值间距8px |

**核心指标卡片设计** (更大更突出):

| 元素 | 规范 |
|-----|------|
| **容器** | 强调卡片，内边距20px，阴影更重 |
| **指标名称** | 14px, `#86909C`, 400字重 |
| **指标数值** | 28px, `#1D2129`, 700字重 |
| **对比差值** | 13px, 根据差值含义着色 |
| **布局** | 垂直居中，名称在上，数值在中，差值在下 |

**对比差值颜色规则**:

根据指标含义，差值颜色代表好坏：

| 指标类型 | 高于均值 | 低于均值 | 等于均值 |
|---------|---------|---------|---------|
| **投诉次数** | 不好 → 红色 `#F44336` | 好 → 绿色 `#4CAF50` | 灰色 `#86909C` |
| **解决率** | 好 → 绿色 `#4CAF50` | 不好 → 红色 `#F44336` | 灰色 `#86909C` |
| **越级次数** | 不好 → 红色 `#F44336` | 好 → 绿色 `#4CAF50` | 灰色 `#86909C` |

**可视化示例**:

```
┌─────────────────────────────┐
│  总投诉次数                   │  ← 14px #86909C
│                             │
│              5              │  ← 28px #1D2129 700
│                             │
│  高于均值 +2                 │  ← 13px #F44336
└─────────────────────────────┘
```

### 5.3 按钮组件

**按钮类型与样式**:

| 按钮类型 | 背景色 | 文字色 | 边框 | 圆角 | 使用场景 |
|---------|-------|-------|------|------|---------|
| **主按钮** | `#1976D2` | `#FFFFFF` | 无 | 8px | 主要操作，如重新分析、重新生成 |
| **次按钮** | `#FFFFFF` | `#1976D2` | `1px solid #1976D2` | 8px | 次要操作，如清空筛选、取消 |
| **文字按钮** | 透明 | `#1976D2` | 无 | 8px | 文字链接型按钮 |

**按钮尺寸规范**:

| 尺寸 | 高度 | 左右内边距 | 字号 |
|-----|------|-----------|------|
| **大按钮** | 48px | 24px | 16px |
| **中按钮** | 40px | 20px | 14px |
| **小按钮** | 32px | 16px | 13px |

**按钮状态规范**:

| 状态 | 效果 |
|-----|------|
| **默认** | 按上述样式 |
| **Hover** | 主按钮: 深蓝 `#0D47A1`；次按钮: 浅蓝背景 `#E3F2FD` |
| **Active** | 进一步压深颜色 |
| **Disabled** | 透明度降低到50%，鼠标不可点击 |

**使用场景示例**:
- AI区块底部的"重新分析"/"重新生成"按钮使用主按钮，中尺寸
- 搜索清空、筛选清除使用次按钮，小尺寸

### 5.4 标签组件 (客服行动标签)

标签用于展示用户风险标签和行动建议，按风险等级着色：

**设计规范**:

| 属性 | 数值 |
|-----|------|
| **形状** | 胶囊形（圆角） |
| **圆角** | 20px |
| **文字** | 14px, 500字重 |
| **上下内边距** | 6px |
| **左右内边距** | 16px |
| **行高** | 1.4 |
| **标签间距** | 水平8px, 垂直8px |

**标签类型按风险等级**:

| 类型 | 背景色 | 文字色 | 示例图标 |
|-----|-------|-------|---------|
| **高危标签** | `#FFEBEE` | `#B71C1C` | 🚨 高危越级预警 |
| **中危标签** | `#FFF3E0` | `#E65100` | 🔥 情绪爆发/极易怒 |
| **关注标签** | `#FFFDE7` | `#F57F17` | ⚠️ 多次重复投诉 |
| **正常标签** | `#E8F5E9` | `#2E7D32` | ✅ 常规跟进处理 |
| **中性标签** | `#F5F7FA` | `#4E5969` | 📶 网络体验受损 |

**布局规范**:
- 标签从高危到正常按顺序排列，高危在前
- 多个标签自动换行，保持8px间距
- 每个标签建议前置emoji图标，增强识别性

## 6. 数据可视化样式设计

### 6.1 柱状图（投诉指标对比）

本产品主要图表为柱状图，用于对比当前用户指标和总体平均值：

**配色方案**:

| 系列 | 颜色 | 说明 |
|-----|------|---------|
| **当前用户** | `#1976D2` | 主色蓝色，突出当前用户 |
| **平均值** | `#E0E0E0` | 灰色，作为对比背景 |

**样式规范**:

| 元素 | 规范 |
|-----|------|
| **图表标题** | 16px, 600, `#272E3B`，底部间距16px |
| **坐标轴文字** | 12px, `#86909C` |
| **网格线** | `#F0F0F0`，虚线，透明度0.8 |
| **柱子宽度** | 自适应，柱子间距保持适中 |
| **柱子圆角** | 4px，顶部圆角 |
| **图例位置** | 顶部，标题右侧 |

**图表容器**:
- 放在卡片容器内，保持内边距20px
- 图表高度固定在350px，确保足够展示空间
- 小屏幕高度自适应调整为300px

### 6.2 数据血缘图（Graphviz可视化）

数据血缘图展示数据来源和加工流程：

**样式优化规范**:

| 元素 | 规范 |
|-----|------|
| **容器** | 白色卡片，标题"🔗 数据血缘追踪"，可折叠 |
| **说明文字** | 折叠状态下显示简短说明文字，帮助用户理解 |
| **节点颜色** | 使用蓝色梯度表示数据处理阶段:<br>- 源数据: `#E3F2FD` 边框 `#1976D2`<br>- 加工处理: `#FFF3E0` 边框 `#FF9800`<br>- 结果输出: `#E8F5E9` 边框 `#4CAF50` |
| **连线颜色** | `#1976D2` 蓝色，箭头指向加工方向 |
| **背景** | 透明，继承卡片背景 |
| **整体缩放** | 自适应容器宽度，支持滚轮缩放 |

**折叠面板设计**:
- 默认折叠，节省空间
- 点击标题区域展开/收起
- 展开图标箭头旋转指示状态

### 6.3 数据可视化设计原则

1. **清晰性优先** - 数据展示应该一目了然，减少不必要的装饰
2. **对比鲜明** - 当前用户数据使用主色突出，平均值使用灰色弱化
3. **一致性** - 色彩使用与整体设计系统保持一致
4. **足够空间** - 图表保持足够尺寸，保证可读性
5. **辅助说明** - 提供必要的标题和说明文字，帮助用户理解
