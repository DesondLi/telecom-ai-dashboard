## 1. 设计概述

### 1.1 设计目标

基于PRD需求，对现有电信AI数据治理系统进行UX优化：

1. **重构信息架构** - 将统计概览从侧边栏移至主内容区顶部，释放侧边栏空间，优化信息层级
2. **优化交互效率** - 添加用户搜索功能，支持筛选，减少操作步骤
3. **强化视觉层次** - 根据风险等级提供清晰的视觉提示，突出关键信息
4. **改善响应式体验** - 适配不同屏幕尺寸
5. **完善状态设计** - 提供清晰的加载、空、错误状态反馈

### 1.2 现有问题分析

根据PRD和代码分析，当前系统存在以下问题需要解决：

| 问题类型 | 具体问题 | 解决方案 |
|---------|---------|---------|
| 信息架构 | 侧边栏内容过多，占用主空间 | 统计概览移至顶部，侧边栏只保留控制区 |
| 用户选择 | 下拉框无搜索，大量用户选择困难 | 添加搜索功能支持快速定位 |
| 风险提示 | 高危用户无明显视觉提示 | 顶部风险警示条，按等级着色 |
| AI交互 | 需要手动点击触发，操作冗余 | 默认自动展示AI分析结果 |
| 视觉层次 | 区块间距不足，内容拥挤 | 优化padding/margin/shadow |
| 标签展示 | 风险区分度不够 | 增大标签尺寸，强化颜色区分 |

### 2.2 侧边栏布局设计

**新侧边栏结构：**

| 区域 | 内容 | 说明 |
|-----|------|------|
| **品牌区** | Logo + 系统名称 | 提供品牌识别，保留电信蓝色主题 |
| **数据源选择** | 单选按钮 | "增强版宽表" / "模拟数据演示"，保持现有功能 |
| **用户搜索** | 搜索输入框 + 过滤下拉 | P0需求：支持快速搜索定位用户 |
| **用户筛选 (P1)** | 多条件筛选器 | - 风险等级：全部/高危/中危/低危<br>- 客户等级：全部/钻石/金卡/银卡/普通<br>- 投诉次数范围：滑块选择 |
| **最近浏览 (P2)** | 最近访问用户列表 | 支持快速切换，最多显示5条 |
| **收藏用户 (P1)** | 收藏用户列表 | 点击星标收藏，快速访问 |
| **帮助信息 (P2)** | 折叠面板 | 功能说明和使用提示 |

**侧边栏宽度**: 固定280px，在小屏幕(<768px)可折叠

### 2.3 主内容区域排版

**区块划分从上到下：**

1. **风险警示条** - 全宽，高度48px，根据风险等级背景色不同
2. **锚点导航栏** - 全宽，背景白色，圆角阴影，提供快速跳转
3. **整体统计概览** - 5列响应式卡片网格
4. **核心指标区** - 4列响应式卡片网格
5. **双列内容区** - 在>1200px屏幕保持双列，中小屏幕变为单列
   - 左：投诉指标详情柱状图
   - 右：客服行动标签 + 业务信息
6. **AI分析区** - 双列布局（>992px），中小屏幕单列
   - 左：AI投诉诊断
   - 右：AI用户画像
7. **可折叠扩展区** - 两个独立可折叠面板
   - 数据血缘图可视化
   - 原始JSON数据查看

### 2.4 信息层级设计

**视觉层级从高到低：**

```
1. 风险警示条               (最高优先级，颜色冲击)
2. 核心指标数值             (大字体，粗体，卡片突出)
3. AI分析结果标题           (次标题，区分于普通内容)
4. 区块标题                 (中号字体，粗体)
5. 普通内容文本             (常规字重，标准大小)
6. 辅助信息/注释            (小号字体，浅色)
```

**间距规范：**

- 页面整体边距: `1rem` (16px)
- 区块间距: `1.5rem` (24px)
- 区块内边距: `1.25rem` (20px)
- 卡片间距: `1rem` (16px)
- 文字行高: 1.6 倍

## 3. 用户交互流程设计

### 3.1 用户选择流程优化

**原有流程:**
```
进入系统 → 下拉框滚动找用户 → 选择用户 → 点击AI按钮 → 查看结果
```

**优化后流程:**
```
进入系统 → 输入搜索关键词 → 点击匹配结果 → 页面自动加载完整分析
                                 ↓
                        (AI结果自动展示，无需额外点击)
```

**搜索交互设计:**

1. **即时搜索**: 输入关键词即时过滤用户列表
2. **匹配高亮**: 在下拉列表中高亮匹配的关键词
3. **键盘导航**: 支持上下箭头选择，回车确认
4. **空结果状态**: 当无匹配用户时显示友好提示，提供清空搜索按钮
5. **保留筛选**: 当应用筛选条件(P1)后，搜索只在筛选结果内进行

**筛选交互设计 (P1):**

- 多条件组合筛选
- 每个筛选条件独立，支持清除单个条件
- 提供"清除所有筛选"按钮
- 筛选结果实时更新用户列表

### 3.2 操作反馈设计

#### 加载状态

| 场景 | 反馈设计 |
|-----|---------|
| 数据源初始加载 | 全屏骨架屏，显示"正在加载数据..."，底部显示加载进度 |
| 用户切换数据 | 内容区域渐变遮罩 + 顶部Spinner，保留侧边栏可见 |
| AI分析生成中 | AI区块内显示Spinner + "AI正在分析，请稍候..."文字，按钮禁用 |
| 图表渲染 | 图表区域占位骨架，直到渲染完成 |

#### 错误状态

| 场景 | 反馈设计 |
|-----|---------|
| 数据源文件不存在 | 红色错误卡片，清晰说明问题，提供检查文件路径提示，不阻止页面运行 |
| API调用失败 | 降级使用本地生成分析结果，顶部显示警告提示"API调用失败，使用本地分析" |
| 用户数据异常 | 显示"该用户数据异常，请尝试选择其他用户"，提供刷新按钮 |
| 网络超时 | 显示"网络请求超时，请检查网络后重试"，提供重试按钮 |

#### 空状态

| 场景 | 反馈设计 |
|-----|---------|
| 搜索无匹配用户 | 图标 + "未找到匹配的用户" + "请尝试其他关键词" + "清空搜索"按钮 |
| 筛选结果为空 | 图标 + "当前筛选条件下没有匹配的用户" + "清空筛选条件"按钮 |
| 收藏列表为空 | 提示"点击用户详情页的星标可收藏常用用户" |

#### 成功状态

- 数据源加载成功：绿色小提示"已加载 X 个用户"，保持3秒后自动消失
- AI分析完成：直接展示结果，不需要额外成功提示

### 3.3 AI分析结果展示设计

**P0需求**: AI分析结果默认展示，无需手动点击

**布局设计:**

```
┌───────────────────────────────────┐
│  🤖 AI 智能分析结果                │
│                                   │
│  ┌──────────────┬──────────────┐  │
│  │ 🔍 投诉诊断   │  📝 用户画像  │  │
│  │              │              │  │
│  │ 自动分析结果  │ 自动生成报告 │  │
│  │              │              │  │
│  │ [重新分析]按钮 │ [重新生成]按钮 │ │
│  └──────────────┴──────────────┘  │
└───────────────────────────────────┘
```

**视觉区分**:
- AI结果区域使用浅蓝色边框 (`#E3F2FD`)
- 轻微浅蓝色背景 (`#FAFCFE`)
- 与普通内容区块形成视觉区分
- 保持圆角和阴影层次

**重新分析功能**:
- 每个AI区块底部保留"重新分析"按钮
- 点击后重新调用LLM生成新的分析结果
- 支持用户获取不同角度的分析建议

### 3.4 响应式适配方案

**断点设计:**

| 断点范围 | 布局适配方案 |
|---------|-------------|
| **≥ 1920px** (大屏桌面) | 侧边栏280px固定，主内容充分利用空间，保持所有双列布局 |
| **1200px - 1919px** (普通桌面) | 侧边栏280px固定，保持双列布局，适当缩小内边距 |
| **768px - 1199px** (平板) | 侧边栏280px固定，双列布局变为单列，所有区块全宽 |
| **< 768px** (手机移动端) | 侧边栏默认折叠，点击汉堡按钮展开，所有内容单列展示 |
| **< 360px** (小屏手机) | 进一步缩小边距和字体，保证核心内容可阅读 |

**网格适配规则:**
- 统计概览 (5列):
  - ≥1200px: 5列
  - 768-1199px: 3列
  - <768px: 2列
- 核心指标 (4列):
  - ≥1200px: 4列
  - 768-1199px: 2列
  - <768px: 2列
- AI分析区 (2列):
  - ≥992px: 双列
  - <992px: 单列

## 5. 交互规范说明

### 5.1 色彩规范

**主色调 (电信蓝):**
- 主色: `#1976D2` (深蓝色) - 品牌、主按钮、标题
- 浅色: `#BBDEFB` (浅蓝色) - 背景、边框、标签
- 深色: `#0D47A1` (深蓝黑) - 强调文字

**风险等级色彩:**
| 等级 | 背景色 | 文字色 | 使用场景 |
|-----|-------|-------|---------|
| 高危 | `#FFEBEE` | `#B71C1C` | 警示条、高危标签 |
| 中危 | `#FFF3E0` | `#E65100` | 警示条、中危标签 |
| 低危/关注 | `#FFFDE7` | `#F57F17` | 警示条、提醒标签 |
| 正常/低风险 | `#E8F5E9` | `#2E7D32` | 警示条、正常标签 |

**中性色:**
- 背景: `#F5F7F9`
- 卡片背景: `#FFFFFF`
- 边框: `#E6E6E6`
- 一级文字: `#1A1A1A`
- 二级文字: `#666666`
- 提示文字: `#999999`

### 5.2 字体规范

- **主标题**: 24px / 700 字重
- **区块标题**: 18px / 600 字重
- **卡片数值**: 28px / 700 字重 (核心指标)
- **正文**: 14px / 400 字重
- **辅助文字**: 12px / 400 字重
- **标签文字**: 14px / 500 字重

### 5.3 卡片设计规范

- **圆角**: 12px
- **阴影**: `0 4px 12px rgba(0,0,0,0.05)`
- **边框**: 可选，浅色边框
- **内边距**: 20px
- **底部间距**: 20px

### 5.4 标签设计规范 (P0-05)

**改进后的标签样式:**
- 圆角: 20px (胶囊形状)
- 左右内边距: 16px
- 上下内边距: 6px
- 文字: 14px 半粗体
- 行高: 1.4
- 按风险等级自动匹配背景色和文字色

示例HTML结构:
```html
<span class="tag tag-critical">🚨 高危越级预警</span>
<span class="tag tag-warning">🔥 情绪爆发/极易怒</span>
<span class="tag tag-success">✅ 常规跟进</span>
<span class="tag tag-neutral">📶 网络体验受损</span>
```

### 5.5 核心指标卡片设计规范 (P0-08)

**设计要点:**
- 大字号展示指标数值 (28px)
- 指标名称在上方，小号文字
- 对比差值在下方，显示相对于平均值的差异
- 差值颜色编码：高于平均值红色(不好)，低于绿色(好)，根据指标含义调整
- 卡片阴影比普通卡片稍重，突出层次

### 5.6 风险警示条设计规范 (P0-04)

**设计要点:**
- 位于主内容最顶部，全宽
- 高度: 48px
- 左侧显示风险图标和文字说明
- 整条背景使用对应风险等级浅色
- 文字使用对应风险等级深色
- 一眼就能识别当前用户风险等级

示例：
```
┌─────────────────────────────────────────────────┐
│ 🔴 🚨 高危用户 - 请优先处理                        │
└─────────────────────────────────────────────────┘
背景: #FFEBEE, 文字: #B71C1C
```

### 5.7 锚点导航设计规范 (P1-04)

**设计要点:**
- 位于风险警示条下方，统计概览上方
- 背景白色，圆角12px，阴影
- 横向排列各区块锚点
- 点击平滑滚动到对应区块
- 当前可视区域的锚点高亮(蓝色文字+下划线)
- 在小屏幕可横向滚动

## 7. 交互流程总图

```
用户进入系统
    │
    ▼
[侧边栏] 加载数据源
    │
    ├─> 加载成功 -> 显示"已加载 X 用户"
    │
    └─> 加载失败 -> 显示错误提示 + 重试按钮
         │
         └─> 点击重试 -> 重新加载

    │
    ▼
[侧边栏] 用户选择
    │
    ├─> 输入搜索关键词 -> 实时过滤用户列表
    │
    ├─> 应用筛选条件 -> 在筛选结果中搜索
    │
    └─> 选择目标用户
          │
          ▼
    [主内容] 加载用户数据
    显示加载状态
          │
          ▼
    1. 显示风险警示条 (按风险等级着色)
    2. 显示所有内容区块
    3. 自动触发AI分析 → 显示加载Spinner
    4. AI分析完成 → 展示结果

          │
          ▼
用户可以:
    • 点击锚点导航 → 平滑滚动到对应区块
    • 折叠/展开 数据血缘和原始数据
    • 点击重新分析 → 重新生成AI结果
    • 切换数据源 -> 重新加载数据
    • 选择其他用户 -> 整体刷新主内容
    • (P1) 点击收藏 -> 添加到收藏列表
    • (P2) 点击最近浏览 -> 快速切换

流程结束
```
